# ADR-010.5: RFC-020.5 Prediction Intelligence Upgrade

**Status:** Accepted (RFC-020.5 Implementation Complete)  
**Date:** 2025-07  
**RFC:** RFC-020.5  

---

## Problem

RFC-020 (ADR-010) implemented a deterministic Prediction Engine with 9 core prediction types across 4 forecast windows. This provided:
- Plateau probability, PR probability, recovery/bodyweight/fatigue forecasting
- Goal ETA computation, MRV risk, deload probability, consistency decay

However, predictions were **opaque** — users could see *what* the engine predicted but not *why*. Missing capabilities:
- **What-if simulation** — "What happens if I sleep more / eat more / deload now?"
- **Explainability** — "Why does the engine think I'll plateau?"
- **Risk assessment** — "How reliable is this prediction?"
- **Counterfactual analysis** — "How would missing a workout affect my progress?"

---

## Decision

Extend the Prediction Engine with four new stateless, deterministic engines:

### 1. Scenario Engine (`PredictionScenarioEngine`)
- **Purpose**: Evaluate "what if I change X?" across 10 predefined interventions
- **Architecture**: Reuses the 8 core engines internally; scenarios are Engine outputs with modified parameters
- **Output**: `ScenarioResult` (8 metric comparisons), `ScenarioRanking` (sorted by net benefit)

### 2. Counterfactual Engine (`CounterfactualEngine`)
- **Purpose**: Answer "how would my predictions change if I modified one behavior?"
- **Architecture**: Single-parameter modification, single-engine re-run
- **Output**: `CounterfactualResult` with delta, percent change, direction, impact level, NL explanation

### 3. Explainability Engine (`ExplainabilityEngine`)
- **Purpose**: Explain *why* a prediction was made
- **Architecture**: Post-hoc analysis of Prediction artifacts (evidence, reasoning, assumptions, risks)
- **Output**: `ExplainabilityDetail` with factor contributions, reason chain, NL + MR explanations

### 4. Risk Engine (`RiskEngine`)
- **Purpose**: Quantify prediction reliability
- **Architecture**: Computes risk from confidence, variance, sample size, volatility, sensitivity
- **Output**: `RiskMetrics` with 5 dimensions + composite risk score and level

### Integration
- All 4 engines are called by `PredictionService.generate_all_predictions()`
- Outputs attached to `PredictionResult` fields (`scenario_results`, `counterfactual_results`, `explainability`, `risk_metrics`, `scenario_rankings`)
- No changes to core engine logic, domain separation, or existing APIs

---

## Alternatives Considered

| Alternative | Reason Rejected |
|---|---|
| **Inline explainability in engines** | Violates SRP; engines predict, they don't explain |
| **Single monolithic analysis engine** | Harder to test, compose, and maintain separately |
| **External ML model for explanations** | Violates determinism principle; unnecessary complexity |
| **UI-only explanations** | Loses structured data for API/export consumers |

---

## Consequences

### Positive
- Predictions are fully explainable (factor ranking, reason chain, NL + MR)
- Users can simulate 10 behavioral interventions
- Risk is quantified per prediction and per forecast
- 159 new tests across all 4 engines (272 total prediction tests)
- Zero regressions in existing prediction, recovery, or gymbrain tests
- All engines are deterministic, stateless, pure — composable by design

### Negative
- `PredictionResult` grows larger (more fields for scenario/counterfactual/explainability/risk data)
- `generate_all_predictions()` is slower due to all scenario and counterfactual evaluations (still sub-second for a single athlete)

### Neutral
- New domain entities added: `ScenarioIntervention` (enum, 10), `CounterfactualType` (enum, 5), `FactorContribution`, `ReasonStep`, `ReasonChain`, `NLExplanation`, `MRExplanation`, `ExplainabilityDetail`, `ScenarioComparison`, `ScenarioResult`, `ScenarioRanking`, `CounterfactualQuery`, `CounterfactualResult`, `RiskMetrics`
- UI layer receives new data but existing dashboard layout unchanged
- 5 new UI widgets (Scenario Comparison, Risk Meter, Explainability, Confidence Breakdown, Reason Tree) added below the 8-card grid

---

## Migration

### For existing code
- `PredictionResult` fields `scenario_results`, `counterfactual_results`, `explainability`, `risk_metrics`, `scenario_rankings` are optional — existing consumers that read `PredictionResult` will not break
- `PredictionService.__init__` unchanged — new engines are created internally
- `generate_all_predictions()` API unchanged — still returns `PredictionResult`

### For new code
```python
result = prediction_service.generate_all_predictions()
for sr in result.scenario_results:
    ...  # Evaluate scenario comparisons
for cr in result.counterfactual_results:
    ...  # Evaluate what-if deltas
for key, detail in result.explainability.items():
    ...  # Access factor contributions, reason chain, NL/MR
for key, rm in result.risk_metrics.items():
    ...  # Access stability, sensitivity, uncertainty, volatility, overall risk
if result.scenario_rankings:
    ...  # Access ranked interventions
```

---

## Compatibility

- **API**: Fully backward-compatible. `PredictionResult` extended, not modified
- **Domain**: New entities in separate namespace; existing entities unchanged
- **Engines**: New engines in separate files; existing engines untouched
- **Infrastructure**: Repository unchanged (new data is in-memory only)
- **UI**: Dashboard extended with new widgets below existing cards; all existing cards remain

---

## Future Extensions

- **Custom scenario parameters** — allow users to define arbitrary deltas (not just the 10 predefined)
- **Compound counterfactuals** — "What if I sleep more AND eat more?"
- **Trend model explanations** — surface which trend model was chosen and why
- **Historical explainability** — compare explanations across time for the same prediction type
- **Risk thresholds** — configurable risk level boundaries per athlete
- **Explainability export** — JSON/Markdown export of full explanation artifacts
- **Scenario presets** — pre-built scenario combinations (e.g., "optimal recovery", "aggressive cut")
