# Forecast Model Reference

## Forecast Point Generation
All forecast points are generated by `_generate_forecast_points()` utility:
- Linear projection from current value using regression slope
- Confidence decays exponentially (`e^(-0.05 * day)`)
- Bounds expand linearly with time (`margin = |value| * 0.1 * (1 + 0.15 * day)`)
- Minimum margin of 5.0 units

## Confidence Scoring
Confidence is derived from the coefficient of variation (CV) of input data:
- CV < 5%: HIGH (0.85)
- CV < 10%: MODERATE (0.70)
- CV < 20%: MODERATE (0.50)
- CV < 35%: LOW (0.35)
- CV >= 35%: VERY_LOW (0.20)

## Forecast Windows
| Window | Points Generated | Use Case |
|--------|-----------------|----------|
| 3d | 3 | Tactical decisions (tomorrow's training) |
| 7d | 7 | Weekly planning |
| 14d | 14 | Bi-weekly trend tracking |
| 28d | 28 | Long-term progression |

## Trend Models
Currently all engines use `LINEAR` regression. Future models (`EXPONENTIAL`, `HOLT_WINTERS`) defined in `TrendModel` enum but not yet implemented.

## Prediction Types
Each prediction type maps to a specific engine and produces:
1. A primary `value` (score, probability, days, kg, etc.)
2. A `probability` (0-1) indicating likelihood
3. 3 `scenarios` (best/expected/worst case)
4. A `Forecast` with point-by-point trajectory
